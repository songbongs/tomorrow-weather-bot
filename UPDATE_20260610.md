# 업데이트 이력 (2026.06.10)

## 1. 대기질 예보 API 전환 및 파싱 최적화
* **기존 문제점:** 저녁 8시 15분에 내일 날씨 브리핑 시, 에어코리아의 실시간 대기 오염도 API를 사용하여 오늘 저녁 8시 기준 실시간 수치를 "내일 대기질" 타이틀 아래 노출하는 기능적 왜곡이 있었습니다.
* **개선 사항:** 
  * 에어코리아 대기질 예보통보 조회 API (`getMinuDustFrcstDspth`)로 호출 대상을 전환하여 내일의 공식 예보 등급(좋음, 보통, 나쁨 등)을 수신하도록 수정했습니다.
  * 입력된 주소지 정보(예: `경기도 용인시 기흥구 구갈동`)를 분석하여 매칭되는 예보 권역(예: `경기남부`, `서울` 등)을 동적으로 판별하는 매핑 유틸리티를 추가했습니다.
  * 루프 내에서 매 위치마다 API를 반복 호출하지 않고, **루프 외부에서 전국 단위 대기질 예보를 단 1회 일괄 수집**한 뒤 매핑하는 구조로 최적화하여 API 트래픽 쿼터 소모를 방지하고 속도를 향상했습니다.
* **수정 파일:**
  * [app.py](app.py)
  * [clients/airkorea_client.py](clients/airkorea_client.py)
  * [services/compare_service.py](services/compare_service.py)
  * [services/message_builder.py](services/message_builder.py)

## 2. 의상 추천 알고리즘 고도화 (절대 온도 기준 도입)
* **기존 문제점:** 오늘과 내일의 상대적인 기온 차이(예: +3.3도)만 고려하여, 한겨울 영하권 한파 날씨(예: 영하 10도에서 영하 7도로 상승)임에도 "오늘보다 아침 기온이 올라 포근할 것"이라는 비현실적이고 모순된 추천 멘트가 생성되었습니다.
* **개선 사항:** 
  * 추천 알고리즘 내에 **절대 온도 기준 구간**(한겨울 최저 5°C / 최고 10°C 이하, 한여름 최고 28°C / 최저 22°C 이상)을 신설했습니다.
  * 겨울철 기온 상승 시 "기온은 오르지만 여전히 한겨울 추위이니 든든히 무장하세요"와 같은 실제 생활 밀착형 멘트를 조합하도록 로직을 강화했습니다.
* **수정 파일:**
  * [services/clothing_advisor.py](services/clothing_advisor.py)
