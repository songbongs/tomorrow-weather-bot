# Tomorrow Weather Briefing Bot

매일 저녁 8시 15분에 내일 날씨와 공기질을 브리핑해 주는 텔레그램 봇 서비스입니다.

## 기존 프로젝트와 차이점
- 아침에 "어제 vs 오늘"을 비교하던 방식에서 벗어나, 저녁에 "오늘 vs 내일"을 미리 비교합니다.
- 복장/우산/마스크 챙기기 등 내일 외출 준비를 돕는 생활 밀착형 알림을 제공합니다.
- 기상청, AirKorea 등 한국 특화 공공 데이터를 활용합니다.

## 기능 특징
- **다지역 지원**: 한 번에 여러 지역의 날씨 정보를 조회하고 순차적으로 브리핑을 전송합니다.
- **안정성**: 한 지역의 조회가 실패하더라도 나머지 지역은 정상적으로 발송합니다. 공기질 API 장애 시 날씨 정보만 발송하는 Fallback 로직을 탑재했습니다.
- **행동 추천 알고리즘**: 오늘과 내일의 아침, 낮, 저녁 기온 차를 계산하여 옷차림을 조언합니다.

## 설치 및 실행 방법

### 1. 레포지토리 클론
```bash
git clone <repository-url>
cd tomorrow-weather-briefing-bot
```

### 2. 패키지 설치
Python 3.11 이상을 권장합니다.
```bash
pip install -r requirements.txt
```

### 3. 환경변수 설정
`.env.example` 파일을 복사하여 `.env` 파일을 생성하고 내용을 채웁니다.
```bash
cp .env.example .env
```

### 4. 텔레그램 봇 생성 및 Chat ID 확인
1. 텔레그램에서 `BotFather`를 검색하여 봇을 생성하고 HTTP API 토큰을 발급받아 `TELEGRAM_BOT_TOKEN`에 넣습니다.
2. 생성한 봇에 메시지를 보낸 후, `https://api.telegram.org/bot<TOKEN>/getUpdates` 를 브라우저나 curl로 열어 `chat_id`를 확인하고 `TELEGRAM_CHAT_ID`에 넣습니다.

### 5. API 키 준비
공공데이터포털(data.go.kr)에서 다음 두 가지 API 활용 신청을 한 후 서비스 키(디코딩)를 환경변수에 넣습니다.
- **기상청_단기예보 ((구)_동네예보) 조회서비스**: `KMA_SERVICE_KEY`
- **한국환경공단_에어코리아_대기오염정보**: `AIRKOREA_SERVICE_KEY`

### 6. 로컬 실행
```bash
python app.py
```

## GitHub Actions 자동 실행 설정

매일 저녁 자동 실행을 위해 `.github/workflows/evening_weather.yml`이 포함되어 있습니다.
GitHub 저장소의 `Settings` -> `Secrets and variables` -> `Actions`에 아래 내용들을 Secret으로 추가하세요.
- `KMA_SERVICE_KEY`
- `AIRKOREA_SERVICE_KEY`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `LOCATIONS_JSON` (또는 `LOCATIONS`)
- `TIMEZONE` (기본값: Asia/Seoul 사용 시 생략 가능)

> **Cron 설정**: 한국시간(KST) 저녁 8시 15분은 UTC 기준 오전 11시 15분이므로 `15 11 * * *` 로 설정되어 있습니다.

## 메시지 예시
📍 **서울특별시 강남구 역삼동**

📅 **오늘 (2026.03.11 (수))**
🌡️ 최고: 6.9°C / 최저: -4.3°C
💧 강수량: 0.0mm (확률: 0%)

📅 **내일 (2026.03.12 (목))**
🌡️ 최고: 7.6°C / 최저: -1.0°C
💧 강수량: 0.0mm (확률: 0%)
🔹 미세먼지: 미세(보통/43), 초미세(나쁨/30)

🗣️ **오늘 vs 내일 비교**
🔹 아침 기온: 오늘보다 3.3도 높아요.
🔹 낮 기온: 오늘보다 1.2도 높아요.
🔹 저녁 기온: 오늘보다 0.2도 낮아요.
🔹 낮에는 오늘보다 조금 더 가볍게 입어도 괜찮아요.
🔹 마스크를 챙기세요. (공기질 나쁨 이상)

## 트러블슈팅 (자주 묻는 질문)

### 1. 🔑 "기상청과 에어코리아 API 키가 똑같이 생겼어요. 정상인가요?"
**네, 정상입니다!** 공공데이터포털은 한 사용자의 계정당 고유한 '일반 인증키(Decoding)'를 하나만 발급합니다. 기상청 데이터를 부르든, 환경공단 데이터를 부르든 똑같은 키 배열을 `.env` 파일의 두 곳에 모두 붙여넣으시면 됩니다. (단, 복사할 때 키 앞뒤로 띄어쓰기 공백이 들어가지 않도록 주의하세요!)

### 2. ⚠️ "기상청 API Error: NO_DATA 오류가 나옵니다."
이 봇은 원래 저녁 8시 15분에 '오후 5시 이후에 최신화된' 기상청 단기예보를 가져오도록 설계되었습니다. 만약 아직 오후 5시가 되지 않은 오전/낮 시간대에 수동으로 테스트(`python app.py`)를 하면 기상청에서 "아직 오늘 저녁 데이터가 없다"라며 에러를 낼 수 있습니다. (현재 버전은 낮에도 가장 최근 데이터를 자동으로 불러오도록 코드가 수정되어 보완되었습니다.)

### 3. 📍 "두 지역 날씨가 똑같이 나오고, 미세먼지는 '정보 없음'으로 나와요."
`.env`의 지역 설정 방식이 잘못되었을 확률이 높습니다. 단순 '문자열' 방식(`LOCATIONS="역삼동, 성복동"`)을 쓰면 시스템이 동네 이름만으로는 정확한 기상청 격자 좌표와 대기오염 측정소를 찾지 못해 임시 기본값을 사용합니다.
반드시 **JSON 방식**을 사용해 각 지역의 정확한 격자 좌표(`nx`, `ny`)와 미세먼지 측정소명(`station`)을 짝지어 입력해야 합니다.
👉 **올바른 예시:** `LOCATIONS_JSON=[{"name":"서울특별시 강남구 역삼동","nx":61,"ny":125,"station":"강남구"}]`

### 4. ⚙️ "GitHub Actions에 내 코드를 다 올렸는데, 왼쪽 메뉴에 시계(Evening Weather Briefing)가 안 나타나요."
바탕화면에서 파일들을 마우스로 드래그 앤 드롭해서 깃허브에 올릴 때, 윈도우 환경에서는 이름이 **점(.)으로 시작하는 `.github` 폴더를 숨김 처리**해서 깃허브에 안 올리는 경우가 많습니다. 
이럴 때는 깃허브 사이트 내 저장소 화면에서 [Add file] -> [Create new file]을 누르고 직접 `.github/workflows/evening_weather.yml` 경로를 타이핑한 뒤 코드를 붙여넣어 주시면 해결됩니다.

### 5. 🚫 "나머지 일반적인 에러들"
* **API Timeout (응답 없음):** 공공 API 서버 장애로 가끔 응답이 늦어질 수 있으며, 이에 대응해 최대 3회 재시도하도록 설정되어 있습니다. 계속 실패하면 봇은 그 지역만 건너뜁니다.
* **텔레그램 발송 실패:** `.env`와 깃허브 Secrets에 넣은 토큰이나 Chat ID가 유효한지 띄어쓰기 없이 잘 들어갔는지 다시 한번 확인해 주시고, 봇 채팅창에서 `/start`를 한 번 눌러놓았는지 반드시 점검하세요.

### 6. 📅 "공공데이터포털 API 만료일(예: 2년 뒤)이 되면 어떻게 되나요?"
만료일이 지나면 API 호출 권한이 사라져 봇이 에러를 내며 멈춥니다. 
**[해결 방법]**
공공데이터포털(data.go.kr) 홈페이지에 로그인 -> 마이페이지 -> 오픈API -> 개발계정에 들어가서 기상청과 에어코리아 API 각각 **[연장신청]** 버튼만 눌러주시면 됩니다.
**💡 중요:** 연장을 해도 기존의 부여된 API 키 값은 똑같이 유지되므로, `.env` 파일이나 깃허브(GitHub Actions) **코드는 단 한 글자도 수정하실 필요가 없습니다.** 연장 버튼 클릭 즉시 다시 예전처럼 정상 작동합니다!
